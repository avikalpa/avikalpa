# AGENTS.md

## Mission

Maintain the public profile repo as a concise front door into Avikalpa Kundu's work: terminal-first tooling, agentic workflows, and self-hosted systems.

## Boundaries

- Keep this repo small and public-facing.
- Prefer links to real repositories over duplicating project docs here.
- Do not turn the profile into a changelog dump or generic portfolio page.

## README And Release Notes Contract

- `README.md` is the public landing page and should stay focused on current work, coherent system themes, and where each active repo fits.
- Keep the top of the page scannable and link outward quickly.
- Any changelog or release note should explain meaningful profile-surface changes such as new flagship projects or major positioning changes, not routine wording tweaks.
- Release pages should reuse curated notes rather than autogenerated text.

## Release Notes Automation

- Keep `CHANGELOG.md` current with `## Unreleased` at the top while work is in flight.
- When cutting a release, move the user-visible notes into an exact version section before or during the tag workstream.
- Release automation should prefer the exact version section and fall back to `Unreleased` so curated notes still publish when the rename is late.
- Release pages should use curated changelog text rather than autogenerated notes.
- Keep profile-release notes sparse and intentional so the repo stays a clean landing page.
